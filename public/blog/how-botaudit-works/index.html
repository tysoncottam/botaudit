Company-by-company breakdown

Typeform (B-) — The best performer, but escalation is a gap

Typeform’s Zendesk bot scored the highest of the group at B- (82/100). It gave consistent, helpful answers on account and billing questions and had the fastest average response times. The one weak spot: when customers asked to speak with a human, the bot didn’t offer a clear escalation path — it redirected to the help center instead.

Recommendation: Add an explicit escalation path. When a customer asks for a human, the bot should confirm that a ticket is being created and set expectations on response time.

Angel (C+) — Good billing answers, inconsistent on edge cases

Angel’s bot handled billing and account questions well, with consistent answers on 7 of 10 questions. It struggled on edge cases and nonsensical input, returning inconsistent error messages across runs. The quality scores were mid-range — responses addressed the question but rarely provided step-by-step guidance.

Squarespace (C+) — Consistent but over-relies on escalation

Squarespace’s bot scored well on consistency — 9 out of 10 questions got the same answer every time. But when it didn’t know the answer, it defaulted to “let’s open a support case” without trying to help first.

Grammarly (C) — Smart AI, but inconsistent wording triggers false alarms

Grammarly uses a Forethought AI bot that gives genuinely helpful answers. The challenge: because it generates fresh responses each time, traditional “exact match” testing would flag every answer as inconsistent. Our semantic similarity analysis correctly identified that 4 of the 7 “inconsistent” answers actually conveyed the same information.

Nature’s Sunshine uses a Zoom Contact Center bot that struggled with 4 out of 10 questions, responding with “Sorry, I didn’t catch that.” It handled password resets and refund policy well, but failed on billing disputes, urgency, and security concerns.

OpenAI (F) — Good answers, but wildly inconsistent

OpenAI’s bot gives substantive, helpful answers — when it works. The problem: it gave a different answer on almost every run. Only the “I need to speak to a human” question got a consistent response (immediate escalation). The bot also requires an email before responding, which adds friction.

Calendly (F) — Bot often ignores the question entirely

Calendly’s Ada-powered bot frequently responded with its greeting (“What can I help you with?”) instead of actually answering the question. The underlying issue: Ada’s chat input uses React controlled inputs that require simulated keystrokes to register properly — simple clipboard paste doesn’t always trigger the submission handler. The bot itself is capable; the submission reliability isn’t.

Pluralsight (F) — Widget doesn’t load for automated browsers

Pluralsight’s Qualified bot returned a score of 12/100 — the lowest in the group. The core issue: Qualified’s routing widget applies anti-bot detection and doesn’t render its chat interface in headless browser environments. In a visible browser it loads correctly, but because the widget doesn’t appear in headless mode, 9 of 10 questions couldn’t be tested. The 12/100 reflects the reliability penalty for bot inaccessibility.

This is actually a useful data point: if your chat widget uses bot-detection that blocks legitimate accessibility tools, it will also block some real users on slower connections or older browsers. Qualified’s routing-only design (no free-text input) also means customers can’t express problems outside the preset routing options.

How we built the grading system

Traditional bot testing tools compare responses with exact string matching. That means an AI bot that says “Here’s how to reset your password” vs “To reset your password, follow these steps” would be flagged as “inconsistent” — even though both answers are correct.

BotAudit uses semantic similarity analysis (token-based cosine similarity) to classify response consistency into four tiers:

  • Identical — Same text every time (knowledge-base bots like Squarespace)
  • Equivalent — Different wording, same information (AI bots like Grammarly)
  • Variable — Same topic, different information provided
  • Contradictory — Conflicting answers (the real problem)

The overall grade combines consistency (40%), response quality (40%), and reliability (20%). Quality is scored 1–5 based on whether the response actually helped, provided actionable steps, or just deflected.
